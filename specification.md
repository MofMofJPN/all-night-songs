## 5. データ設計

### 5.1 タグ定義

#### 気分（mood）— 6種
- `切ない` — 失恋・別れ・寂しさを含む曲
- `エモい` — 青春・感傷・ノスタルジー系
- `元気・前向き` — 明るく背中を押す系
- `盛り上がる` — テンション上がる・ライブ映え系
- `クール・かっこいい` — 洗練・尖り・スタイリッシュ
- `優しい・癒し` — 穏やか・包み込む系

#### テンポ（tempo）— 3種
- `スロー` — バラード〜ゆっくり
- `ミドル` — 標準的なテンポ
- `アップ` — アップテンポ・ノリのいい

#### シーン（scene）— 4種
- `しっとり聴きたい` — 落ち着いて聴く曲
- `一緒に盛り上がりたい` — ライブ・盛り上がり系
- `泣きたい夜` — 感情に浸る・深夜向け
- `朝・お出かけ気分` — 明るい時間帯・移動中

### 5.2 曲データの構造（TypeScript型）

```typescript
interface Song {
  id: string;              // ユニークID (例: "001")
  title: string;           // 曲名
  artist: string;          // アーティスト名
  kanaSection: string;     // 五十音区分: "あ"|"か"|"さ"|"た"|"な"|"は"|"ま"|"や"|"ら"|"わ"
  tags: {
    mood: string[];        // 1〜2個
    tempo: string;         // 1個
    scene: string[];       // 1〜2個
  };
  keyAdjustment?: string;  // キー調整 (例: "-1", "+2") [オプション]
  status?: string[];       // 状態バッジ (例: ["1番のみ"], ["練習中"]) [オプション]
  note?: string;           // 自由メモ (例: "苦しめ🥹") [オプション]
}
```

### 5.3 全曲データ
- 別ファイル `data/songs.json` 参照（156曲）

## 7. マッチングロジック

### 7.1 スコア計算式

ユーザーの3つの答えと各曲のタグを比較:

```
気分タグに含まれる:     +2点（重み大）
テンポタグが一致:       +1点
シーンタグに含まれる:   +1点
─────────────────────
最大スコア:             4点（3軸完全一致）
```

**実装ロジック（疑似コード）:**
```javascript
function calculateScore(song, answers) {
  let score = 0;
  if (song.tags.mood.includes(answers.mood)) score += 2;
  if (song.tags.tempo === answers.tempo) score += 1;
  if (song.tags.scene.includes(answers.scene)) score += 1;
  return score;
}
```

### 7.2 結果表示ルール

1. 全曲のスコアを計算
2. **直前の診断結果（lastResults）に含まれる曲は除外**
3. スコア降順でソート
4. 上位5曲を選出
5. セクション分け:
   - スコア4点 → 「🎯 ピッタリの曲」
   - スコア3点 → 「✨ 近い雰囲気の曲」
   - スコア2点以下 → 「✨ 近い雰囲気の曲」（同セクションでOK）
6. 同点の曲が複数ある場合は**ランダムに並べる**

### 7.3 セッション内重複回避

- 「もう一度診断」を押した場合、**直前に表示した5曲を次回除外**
- 保存先: `sessionStorage` （タブを閉じたらリセット）または `ans:lastResults` （LocalStorage）
- 仕様: LocalStorageに保存し、新規セッションで初期化

### 7.4 0件時の処理

理論上、全曲スコア0は起こりにくいが念のため:

- スコア0の曲しかない場合（除外で全消失等）
- **気分軸のみで再検索**して何かしら5曲表示
- 表示メッセージ: `ピッタリな曲が見つからなかったので、気分が近い曲をご紹介します`

### 8.2 タイポグラフィ

**Google Fonts読み込み:**
- `Zen Maru Gothic` （和文・見出し）
- `Noto Sans JP` （和文・本文）
- `Bebas Neue` または `Oswald` （英文・ロゴ）

**サイズ階層:**
- ロゴ: 32〜48px
- 見出し: 20〜24px
- 本文: 14〜16px
- 補助: 12〜13px

### 8.3 装飾要素

#### ブラケット `[ ]`
リットリンクの装飾を踏襲。見出しやボタンに使用。

```html
<button class="btn-bracket">[ 今夜の1曲を探す ]</button>
```

CSSで `[ ` と ` ]` を疑似要素で実装してもOK。

#### 星のアニメーション
背景に1〜2pxの白い点をランダムに配置、`opacity` を3〜5秒で穏やかに変化。

```css
@keyframes twinkle {
  0%, 100% { opacity: 0.3; }
  50% { opacity: 1; }
}
```

複数のクラスで周期をずらして自然な瞬きを表現。

#### カラスのシルエット
SVGで実装、ヘッダーやフッターに小さく配置。`opacity: 0.4` 程度の控えめさ。

#### ボタン
- 角丸 8〜12px
- 縁取り: `var(--border-soft)`
- ホバー時: 背景色変更＋柔らかいグロー
- アクティブ時: 軽く沈む

#### カード
- 角丸 12〜16px
- 背景: `var(--color-bg-card)`
- パディング 16〜24px
- 影: `var(--shadow-soft)`

### 8.4 アニメーション原則

- **すべて穏やか・ゆっくり**（HSPに配慮）
- ページ遷移: 300〜500ms のフェード
- ボタンホバー: 150〜200ms の色変化
- ローディング: 控えめなスピナー or 星の脈動
- **NG**: 急激な動き、点滅、ストロボ効果、大きなバウンス

### 8.5 レスポンシブ設計

- **スマホファースト**で設計
- ブレークポイント:
  - `<= 480px` スマホ（メインターゲット）
  - `481px 〜 768px` タブレット
  - `>= 769px` PC
- スマホでの最大コンテンツ幅: 100%（パディングのみ）
- PCでの最大コンテンツ幅: 720px 中央寄せ

## 14. Claude Codeでの実装ガイド

### 14.1 推奨進め方
**段階的実装**を強く推奨:

1. Phase 1A で**動くトップページ**を作る
2. 各Phaseごとに動作確認
3. 各Phase完了時に git commit
4. 全Phase完了後、レスポンシブ・アクセシビリティ最終確認

### 14.2 重要な注意点

**やってほしいこと:**
- ✅ スマホ優先で実装
- ✅ HSPに配慮した穏やかなアニメーション
- ✅ アクセシビリティ配慮（キーボード操作・aria属性）
- ✅ 軽量化（外部ライブラリは Google Fonts のみ程度に）
- ✅ コメントは丁寧に（後で曲追加・修正がしやすいように）

**やらないでほしいこと:**
- ❌ 重いフレームワーク（React, Vue等）の導入
- ❌ 派手なアニメーション、点滅、急激な動き
- ❌ 過度なエフェクト（パーティクル多数、3D表現等）
- ❌ 外部APIへの不要な通信
- ❌ トラッキング・広告コード

### 14.3 テスト観点
- LocalStorageが空でも動作するか
- LocalStorage拒否環境でもエラーにならないか
- 端末サイズが変わっても崩れないか（320px幅で確認）
- マッチング結果が妥当か（実際に診断してみる）
- お気に入り・聴いた記録が正しく保存・復元されるか
- エクスポート→削除→インポートで完全に復元できるか

### 14.4 git 設定
- `.gitignore`: `node_modules/`, `.DS_Store`, `*.log`, `.env`
- `README.md` にプロジェクト概要と使い方を記載
- コミット粒度: Phase単位、または機能単位

- GitHub Pages
- リポジトリ名: `all-night-songs` 推奨
- ブランチ: `main`
